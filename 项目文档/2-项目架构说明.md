好，那就按 1️⃣ 用户入口放 Rust（对外 Data Plane） 来做企业级架构。下面我给你一套可直接落地的实现方案 + 关键代码骨架（Rust Axum + Redis 缓存 + Postgres/Prisma 账务）。

⸻

目标架构（企业级正确姿势）

对外 API：Rust Gateway
	•	POST /v1/chat/completions（OpenAI 兼容）
	•	验证用户 Authorization: Bearer sk-or-xxx
	•	不每次查 Postgres：优先 Redis / LRU 缓存
	•	做网关保护：并发、SSE、超时、熔断、fallback、provider 限流
	•	获取 usage（prompt/completion tokens）
	•	原子扣费：写 usage_logs + 扣 user_balances + 写 transactions

Next.js：Control Plane
	•	发放/吊销 API Key
	•	套餐/限额配置
	•	账单/报表/管理后台
	•	同步 key/计划/权限到 Redis（或让 Gateway 拉取）

⸻

1) Key 校验：Redis + LRU + DB 回源

Redis 里存什么（建议）

Key 用 hash 做索引，不要存明文。
	•	Redis key：ak:<sha256(apiKey)>
	•	Redis value（JSON 或 Redis Hash）：
	•	user_id
	•	status
	•	rate_limit_per_min
	•	plan
	•	model_allowlist（可选）
	•	quota_monthly（可选）
	•	expires_at（可选）

TTL：比如 10~60 分钟（吊销时 Next.js 立即删除对应 Redis key）。

校验流程
	1.	Rust 收到 sk-or-xxx
	2.	sha256(key) → 查 ak:<hash>（Redis）
	3.	命中：直接用
	4.	未命中：查 Postgres（api_keys 表，按 keyHash）→ 写回 Redis → 放行

注意：只有在 Redis miss 时才查 DB，DB 压力会极小。

⸻

2) 计费与扣费：必须用数据库事务

一次请求结束后：
	1.	写 usage_logs
	2.	更新 user_balances.balance -= cost（余额不足则回滚）
	3.	写 transactions（type=usage, amount=-cost）

✅ 这三步必须在一个事务里，否则会出现“扣了钱没记账/记账没扣钱”的对账灾难。

⸻

3) SSE/流式下如何拿到 usage（关键）

情况 A：上游 provider 在响应里给 usage（非 stream 或 stream end 给）
	•	非 stream：直接解析 JSON usage
	•	stream：很多 provider 不会在每个 chunk 给 usage，通常在最后给一个 summary 或需要额外统计

情况 B：自己统计（通用方案）
	•	prompt_tokens：用 tokenizer 估算（可接受误差）或调用 provider 的 token 计数能力（如果有）
	•	completion_tokens：对输出增量进行估算（对企业计费建议尽量准确）
	•	企业级做法：优先使用 provider 官方返回 usage；没有则使用 tokenizer（如 tiktoken 类似实现）。Rust 里可以先“粗估 + 标记 estimated=true”，后续再异步校正也行。

实操建议：第一版先支持 OpenAI：大多数时候 non-stream 会返回 usage；stream 你先“结束后补打一发计费估算”，或者让客户端默认 non-stream（企业客户多数能接受）。

use uuid::Uuid;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ApiKeyMeta {
    pub user_id: Uuid,
    pub status: String, // active/disabled/expired
    pub rate_limit_per_min: i32,
    pub plan: String,
    pub expires_at: Option<String>,
}


⸻

从 Authorization 提取 Key + SHA256

use sha2::{Sha256, Digest};

fn extract_bearer(auth: &str) -> Option<String> {
    let auth = auth.trim();
    if !auth.to_lowercase().starts_with("bearer ") { return None; }
    Some(auth[7..].trim().to_string())
}

fn sha256_hex(s: &str) -> String {
    let mut hasher = Sha256::new();
    hasher.update(s.as_bytes());
    hex::encode(hasher.finalize())
}


⸻

Redis 读取/写入（缓存 key meta）

use redis::AsyncCommands;

async fn redis_get_keymeta(
    conn: &mut redis::aio::MultiplexedConnection,
    key_hash: &str
) -> redis::RedisResult<Option<ApiKeyMeta>> {
    let redis_key = format!("ak:{key_hash}");
    let v: Option<String> = conn.get(redis_key).await?;
    Ok(v.and_then(|s| serde_json::from_str(&s).ok()))
}

async fn redis_set_keymeta(
    conn: &mut redis::aio::MultiplexedConnection,
    key_hash: &str,
    meta: &ApiKeyMeta,
    ttl_seconds: usize
) -> redis::RedisResult<()> {
    let redis_key = format!("ak:{key_hash}");
    let payload = serde_json::to_string(meta).unwrap();
    let _: () = conn.set_ex(redis_key, payload, ttl_seconds).await?;
    Ok(())
}


⸻

DB 回源：按 keyHash 查 api_keys（只在 Redis miss）

你 DB 里应该存 api_keys.key_hash（hash 后的值），不要存明文 key。

use sqlx::PgPool;

async fn db_load_keymeta(pool: &PgPool, key_hash: &str) -> sqlx::Result<Option<ApiKeyMeta>> {
    // 假设表：api_keys(user_id uuid, key_hash text, status text, rate_limit_per_min int, ...)
    let row = sqlx::query!(
        r#"
        SELECT user_id, status, rate_limit_per_min
        FROM api_keys
        WHERE key_hash = $1
        "#,
        key_hash
    ).fetch_optional(pool).await?;

    Ok(row.map(|r| ApiKeyMeta {
        user_id: r.user_id,
        status: r.status,
        rate_limit_per_min: r.rate_limit_per_min,
        plan: "pro".to_string(),         // 可从 subscriptions/plan 表查出并缓存
        expires_at: None,
    }))
}

企业级建议：把 plan/模型权限/额度 也一起查出来并缓存到 Redis，避免每次跨表查询。

5) 对外接口：/v1/chat/completions（网关入口）

核心步骤：
	1.	bearer → hash → Redis → (miss) DB
	2.	状态校验（active）
	3.	保护限流（网关层）
	4.	转发 provider（用平台 OPENAI_API_KEY）
	5.	解析 usage
	6.	事务扣费 & 记账
	7.	返回给用户

下面先给你非 stream 的最小版本（先跑通企业级计费闭环）：

use axum::{Router, routing::post, extract::{State}, http::{HeaderMap, StatusCode}, Json};
use serde_json::Value;
use std::sync::Arc;

#[derive(Clone)]
struct AppState {
    redis: redis::aio::MultiplexedConnection,
    db: sqlx::PgPool,
    openai_key: String,
}

async fn chat_completions(
    State(state): State<Arc<AppState>>,
    headers: HeaderMap,
    Json(body): Json<Value>,
) -> Result<Json<Value>, (StatusCode, String)> {
    let auth = headers.get("authorization").and_then(|v| v.to_str().ok())
        .ok_or((StatusCode::UNAUTHORIZED, "missing authorization".into()))?;
    let api_key = extract_bearer(auth).ok_or((StatusCode::UNAUTHORIZED, "invalid authorization".into()))?;

    // user key -> hash
    let key_hash = sha256_hex(&api_key);

    // Redis lookup
    let mut redis = state.redis.clone();
    let mut meta = redis_get_keymeta(&mut redis, &key_hash).await
        .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "redis error".into()))?;

    if meta.is_none() {
        meta = db_load_keymeta(&state.db, &key_hash).await
            .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "db error".into()))?;

        if let Some(ref m) = meta {
            // cache 30 min
            let _ = redis_set_keymeta(&mut redis, &key_hash, m, 1800).await;
        }
    }

    let meta = meta.ok_or((StatusCode::UNAUTHORIZED, "invalid api key".into()))?;
    if meta.status != "active" {
        return Err((StatusCode::FORBIDDEN, "api key not active".into()));
    }

    // TODO: 网关保护限流（并发/semaphore、provider 并发限制等）

    // Forward to OpenAI (platform key)
    let client = reqwest::Client::new();
    let resp = client
        .post("https://api.openai.com/v1/chat/completions")
        .bearer_auth(&state.openai_key)
        .json(&body)
        .send()
        .await
        .map_err(|_| (StatusCode::BAD_GATEWAY, "provider request failed".into()))?;

    let status = resp.status();
    let v: Value = resp.json().await.map_err(|_| (StatusCode::BAD_GATEWAY, "provider json parse failed".into()))?;

    if !status.is_success() {
        return Err((StatusCode::BAD_GATEWAY, v.to_string()));
    }

    // Extract usage
    let usage = v.get("usage").cloned().unwrap_or(Value::Null);
    let prompt_tokens = usage.get("prompt_tokens").and_then(|x| x.as_i64()).unwrap_or(0) as i32;
    let completion_tokens = usage.get("completion_tokens").and_then(|x| x.as_i64()).unwrap_or(0) as i32;
    let total_tokens = usage.get("total_tokens").and_then(|x| x.as_i64()).unwrap_or(0) as i32;

    // Lookup pricing (cache 也可上 Redis)
    let pricing = sqlx::query!(
        r#"SELECT input_price, output_price FROM model_pricing WHERE model_name = $1"#,
        body.get("model").and_then(|m| m.as_str()).unwrap_or("gpt-4o")
    ).fetch_one(&state.db).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "pricing not found".into()))?;

    // cost: (tokens/1000)*price
    use bigdecimal::{BigDecimal, FromPrimitive};
    let input_price = pricing.input_price;
    let output_price = pricing.output_price;

    let pt = BigDecimal::from_i32(prompt_tokens).unwrap();
    let ct = BigDecimal::from_i32(completion_tokens).unwrap();
    let k = BigDecimal::from_i32(1000).unwrap();

    let cost = (&pt / &k) * input_price + (&ct / &k) * output_price;

    // Atomic billing transaction: usage_logs + balance update + transaction record
    bill_in_tx(&state.db, meta.user_id, key_hash.clone(), body.get("model").and_then(|m| m.as_str()).unwrap_or("unknown"), prompt_tokens, completion_tokens, total_tokens, &cost)
        .await
        .map_err(|e| e)?;

    Ok(Json(v))
}

事务扣费实现（关键）：

async fn bill_in_tx(
    pool: &sqlx::PgPool,
    user_id: uuid::Uuid,
    _key_hash: String,
    model: &str,
    input_tokens: i32,
    output_tokens: i32,
    total_tokens: i32,
    cost: &bigdecimal::BigDecimal,
) -> Result<(), (axum::http::StatusCode, String)> {
    let mut tx = pool.begin().await.map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "tx begin failed".into()))?;

    // 1) Ensure balance row exists
    sqlx::query!(
        r#"INSERT INTO user_balances (user_id, balance) VALUES ($1, 0)
           ON CONFLICT (user_id) DO NOTHING"#,
        user_id
    ).execute(&mut *tx).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "ensure balance failed".into()))?;

    // 2) Check balance >= cost
    let bal = sqlx::query!(
        r#"SELECT balance FROM user_balances WHERE user_id = $1 FOR UPDATE"#,
        user_id
    ).fetch_one(&mut *tx).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "balance read failed".into()))?;

    if bal.balance < *cost {
        tx.rollback().await.ok();
        return Err((StatusCode::PAYMENT_REQUIRED, "insufficient balance".into()));
    }

    // 3) Insert usage log
    sqlx::query!(
        r#"
        INSERT INTO usage_logs (user_id, model, input_tokens, output_tokens, total_tokens, cost, status, created_at)
        VALUES ($1, $2, $3, $4, $5, $6, 'success', NOW())
        "#,
        user_id,
        model,
        input_tokens,
        output_tokens,
        total_tokens,
        cost.clone(),
    ).execute(&mut *tx).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "usage log insert failed".into()))?;

    // 4) Deduct balance
    sqlx::query!(
        r#"UPDATE user_balances SET balance = balance - $1, updated_at = NOW() WHERE user_id = $2"#,
        cost.clone(),
        user_id
    ).execute(&mut *tx).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "balance update failed".into()))?;

    // 5) Transaction record (negative for usage)
    let neg_cost = -cost.clone();
    sqlx::query!(
        r#"
        INSERT INTO transactions (user_id, amount, type, description, created_at)
        VALUES ($1, $2, 'usage', $3, NOW())
        "#,
        user_id,
        neg_cost,
        format!("model={model}, tokens={total_tokens}")
    ).execute(&mut *tx).await
      .map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "transaction insert failed".into()))?;

    tx.commit().await.map_err(|_| (StatusCode::INTERNAL_SERVER_ERROR, "tx commit failed".into()))?;
    Ok(())
}

这就是企业级“不会账错”的核心闭环。

6) 关于“限流”放哪（你选 1 的情况下）

入口在 Rust 后：
	•	业务配额（套餐 token 额度/模型权限）：可以由 Redis 缓存 meta 直接判断（不需要 Next.js 在链路上）
	•	网关保护（并发、连接数、provider 并发）：Rust 用 semaphore/令牌桶

所以你可以做到：用户直连 Rust，仍然具备企业级配额控制。

7) Next.js Control Plane 需要做的最少两件事
	1.	创建/吊销 key 时：
	•	写 Postgres api_keys.key_hash
	•	同步写/删 Redis ak:<hash>（立即生效）
	2.	更新 plan/权限/限额时：
	•	更新 Postgres
	•	同步更新 Redis（或设置短 TTL 让它自然刷新）
